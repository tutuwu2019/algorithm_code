# 存储哈希

> 这块听的少。专门为了存储密码而设计的特殊哈希算法。特点：故意设计得很慢(消耗大量计算资源)，以抵御暴力破解。

1. PBKDF2
> 通过多次重复哈希运算来增加计算成本。比较老的标准，但依然可用。

2. bcrypt
> 基于Blowfish 加密算法，内置盐值，具有可调节的"工作因子"(迭代次数),能有效抵抗CPU/ASIC攻击。


3. scrypt
> 不仅计算成本高、而且内存需求也很大，这进一步增加了大规模硬件并行破解速度。有些加密货币就会使用它作为挖矿算法。

4. Argon2
> 目前公认的最佳选择。赢得了2015年密码哈希竞赛。可灵活配置时间成本、内存成本和并行计算成本，安全性最高。现代系统密码存储的推荐算法。
